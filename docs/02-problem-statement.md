# Problem Statement

## Current Challenges

### Fragmented Context

Business and technical context is scattered across:

* Source code
* Git commits
* Jira
* Confluence

### Knowledge Silos

Critical architectural knowledge exists primarily in senior engineers.

### Impact Uncertainty

Developers struggle to understand:

* Downstream impact
* Ripple effects
* Regression risks

### Slow Onboarding

Understanding unfamiliar modules requires significant manual investigation.

## Root Cause

The problem is not code discovery.

The problem is context discovery.

Developers lack a unified view of:

* What a file does
* Why it was changed
* Who changed it
* Which business requirement drove the change
* What may break if modified
